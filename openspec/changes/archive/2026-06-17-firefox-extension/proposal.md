## Why

The current "Simple Tab Manager" extension is built for Chrome Manifest V3 but lacks compatibility with Firefox. Supporting Firefox allows users of Mozilla's browser to use the extension, expanding the user base.

## What Changes

- Add a Firefox-compatible manifest (handling `browser_specific_settings.gecko.id` and `background.scripts`).
- Introduce the `webextension-polyfill` to seamlessly support `browser.*` Promise-based APIs across both browsers, replacing the Chrome-specific `chrome.*` callbacks/promises.
- Update the Google OAuth identity flow to handle Firefox's specific redirect URLs generated by `identity.getRedirectURL()`.
- Implement a simple build/packaging process to output browser-specific bundles if a unified manifest is not feasible.

## Capabilities

### New Capabilities
- `firefox-compatibility`: The extension will run on Firefox (MV3) with full functionality matching the Chrome version.

### Modified Capabilities
- None (there are no existing specs yet).

## Impact

- `manifest.json`: Will require `browser_specific_settings` and potentially a split or script to handle Firefox vs Chrome background declarations.
- Extension Scripts (`background.js`, `popup.js`, `options.js`): Will need to use `browser.*` instead of `chrome.*` and include the polyfill.
- OAuth Credentials: The Google Cloud Console will need the Firefox extension's redirect URL added to its authorized redirect URIs.
- Dependencies: Need to add `webextension-polyfill`.
